## 📘 Описание проекта

Третья часть дипломного проекта курса **«Инженер по тестированию: от новичка до автоматизатора»**.  
Разработаны **автотесты для REST API** учебного сервиса космической бургерной **Stellar Burgers**.

**🎯 Цель:** покрыть тестами ключевые эндпоинты: регистрация, авторизация, создание заказов — с проверкой позитивных и негативных сценариев.

---

## 🧪 Тестовые сценарии

### 👤 Регистрация пользователя (`POST /api/auth/register`)

| Сценарий | Описание | Ожидаемый результат | Статус |
|----------|----------|---------------------|--------|
| ✅ Уникальный пользователь | Создание пользователя с новыми `email` и `name` | `200 OK`, возвращает `accessToken`, `refreshToken`, `user` | ✅ |
| ❌ Дубликат пользователя | Попытка создать пользователя с существующим `email` | `403 Forbidden`, сообщение об ошибке | ✅ |
| ❌ Отсутствие обязательного поля | Создание пользователя без `email` или `name` | `400 Bad Request`, ошибка валидации | ✅ |

---

### 🔐 Авторизация пользователя (`POST /api/auth/login`)

| Сценарий | Описание | Ожидаемый результат | Статус |
|----------|----------|---------------------|--------|
| ✅ Успешный вход | Авторизация существующего пользователя | `200 OK`, возвращает токены и данные пользователя | ✅ |
| ❌ Неверные данные | Вход с неправильным `email` или `password` | `401 Unauthorized`, сообщение об ошибке | ✅ |

---

### 📦 Создание заказа (`POST /api/orders`)

| Сценарий | Описание | Ожидаемый результат | Статус |
|----------|----------|---------------------|--------|
| ✅ С авторизацией | Создание заказа с валидным `accessToken` | `200 OK`, возвращает данные заказа с `track` | ✅ |
| ❌ Без авторизации | Попытка создать заказ без токена | `401 Unauthorized` | ✅ |
| ✅ С ингредиентами | Заказ с валидным массивом `ingredients` | `200 OK`, заказ создан | ✅ |
| ✅ Без ингредиентов | Заказ с пустым массивом `ingredients` | `200 OK`, заказ создан | ✅ |
| ❌ Неверный хеш ингредиента | Заказ с несуществующим `ingredient` hash | `400 Bad Request`, ошибка валидации | ✅ |

## ⚙️ Запуск тестов

### 1️⃣ Клонировать репозиторий

~~~bash
git clone https://github.com/ViktorProkopovich/Diplom_2_QA_Python_YaPracticum.git
cd Diplom_2_QA_Python_YaPracticum
~~~

---

### 2️⃣ Создать и активировать виртуальное окружение

~~~bash
python -m venv venv
~~~

**Активация:**

🪟 **Windows**  
~~~bash
venv\Scripts\activate
~~~

🐧 **MacOS / Linux**  
~~~bash
source venv/bin/activate
~~~

---

### 3️⃣ Установить зависимости

~~~bash
pip install -r requirements.txt
~~~

---

### 4️⃣ Запустить тесты

~~~bash
# Запуск всех тестов с подробным выводом
pytest -v

# Запуск с генерацией отчёта Allure
pytest -v --alluredir=reports/allure_results
~~~

---

### 5️⃣ Просмотреть отчёт о тестировании

~~~bash
# Запуск Allure-сервера для просмотра отчёта
allure serve reports/allure_results
~~~

> 🌐 Отчёт откроется в браузере по адресу `http://localhost:54321`

## 🧠 Что покрывают тесты

| Категория | Проверки |
|------------|-----------|
| **Регистрация** | Уникальный пользователь, дубликат, отсутствие обязательных полей |
| **Авторизация** | Успешный вход, неверные учётные данные |
| **Создание заказа** | С авторизацией/без, с ингредиентами/без, валидный/невалидный хеш |
| **Валидация ответов** | Структура ответа, коды статусов, сообщения об ошибках |
| **Работа с токенами** | Передача `accessToken` в заголовках, обработка просроченных токенов |

## 🧩 Используемые технологии

- 🐍 **Python 3.10+** — язык программирования  
- 🧪 **Pytest** — фреймворк для тестирования  
- 🌐 **Requests** — отправка HTTP-запросов к API  
- 📊 **Allure Report** — визуализация результатов тестирования  
- 🔁 **Параметризация** (`@pytest.mark.parametrize`) — покрытие сценариев данными  
- 🎭 **Фикстуры** (`conftest.py`) — подготовка и очистка тестовых данных  
- 🔐 **Авторизация** — работа с `accessToken` / `refreshToken`
